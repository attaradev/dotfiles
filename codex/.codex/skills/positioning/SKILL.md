# Product Positioning

Use this skill to define a true, defensible position in the market.

## Workflow

1. Write the positioning statement for the ICP.
2. Define differentiated value and category.
3. Build the message hierarchy from headline to CTA.
4. Attach proof points to each claim.
5. List claims to avoid because they are generic or unverifiable.

## Quality rules

- Positioning must be true.
- Avoid generic claims like "easy to use" or "powerful".
- Choose the category deliberately.
- Every message claim needs a proof point.
