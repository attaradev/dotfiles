# User Story Generation

Use this skill to break a feature or workflow into testable user stories.

## Workflow

1. Identify the user types, happy path, and failure modes.
2. Write stories at the right granularity.
3. Include acceptance criteria and edge cases for each story.
4. Follow `references/user-story-format.md`.

## Quality rules

- Each story should deliver standalone value.
- Acceptance criteria must be testable.
- Cover happy path, validation failure, empty state, and permission edge cases.
- Flag stories that need more information.

## Resources

- `references/user-story-format.md` provides story syntax and acceptance criteria guidance.
