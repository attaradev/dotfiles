# Hypothesis & Experiment Design

Use this skill to isolate the riskiest assumption and define the cheapest useful test.

## Workflow

1. State the belief being tested.
2. Identify the riskiest assumption.
3. Design the cheapest experiment that can fail.
4. Set success criteria before running the test.
5. Explain what a negative result means for the idea.

## Quality rules

- Test one assumption at a time.
- Make the test capable of producing a negative result.
- Define success before running the experiment.
- Distinguish learning goals from business goals.
