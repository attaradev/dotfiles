# Technical Blog Post

Use this skill to write a publication-ready technical article for a technical audience.

## Workflow

1. Read the topic and any relevant code or docs.
2. Shape the post around the problem, the solution, and the results.
3. Include code snippets, diagrams, or data only when they add clarity.
4. Follow `references/tech-blog-guide.md`.

## Quality rules

- Lead with the problem.
- Back every improvement claim with a number.
- Keep code snippets minimal.
- Write in a direct, conversational tone.
