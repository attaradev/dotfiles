# Execution Plan

Use this skill to turn a goal into a concrete implementation plan.

## Workflow

1. Read the repository context before planning.
2. Clarify the goal, constraints, and definition of done.
3. Compare two or three viable approaches.
4. Recommend one approach directly and break the work into phases.
5. Include verification, rollback notes, risks, and open questions.

## Output expectations

- Lead with the problem statement.
- Present real alternatives, not a single-option plan.
- Keep phases small enough to validate independently.
- Call out assumptions that may change the plan.
