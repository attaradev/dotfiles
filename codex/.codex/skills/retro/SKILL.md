# Retrospective

Use this skill to prepare a blameless retro with a clear format, timebox, and action item capture.

## Workflow

1. Pick the most appropriate retro format for the situation, or use the format the user specified.
2. Seed the retro with concrete events from recent work when git context is available.
3. Produce a timed facilitation guide and prompt cards for the chosen format.
4. Include a warm-up, action-item template, and closing questions.
5. End with owned follow-up items, not vague improvement wishes.

## Quality rules

- Timebox every section.
- Keep prompts open-ended and blameless.
- Limit action items to concrete, owned commitments with due dates.
- Close with a retro-on-the-retro so the next session improves.
