# Jobs-to-be-Done Analysis

Use this skill to uncover the real progress users are trying to make.

## Workflow

1. State the core job in canonical form.
2. Map the eight stages of the job.
3. Separate functional, social, and emotional dimensions.
4. Write measurable outcome statements.
5. Identify competing solutions and key insights.
6. Save the output to `docs/jtbd-<slug>.md` when useful.

## Quality rules

- Describe progress, not features.
- Make outcome statements measurable.
- Surface at least one non-obvious insight.
- Ask "why" until you hit bedrock.
