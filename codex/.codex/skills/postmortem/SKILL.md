# Post-Incident Postmortem

Use this skill to document an incident without blame and with actionable follow-up work.

## Workflow

1. Read the incident description and available evidence.
2. Build the timeline, root cause analysis, contributing factors, and action items.
3. Keep the tone blameless and systemic.
4. Follow `references/postmortem-template.md`.

## Quality rules

- Root cause must be systemic, not personal.
- Timeline must be chronological.
- Action items must be owned and time-bounded.
- Avoid speculation when evidence is missing.

## Resources

- `references/postmortem-template.md` provides the incident postmortem structure and 5-whys guidance.
