# Ideal Customer Profile Guide

## ICP vs persona

| | ICP | Persona |
|--|-----|---------|
| **Describes** | The company or account (B2B) or user segment (B2C) | The individual buyer or user within that account |
| **Used for** | Sales qualification, marketing targeting, product focus | Messaging, onboarding, feature design |
| **Level** | Account-level | Person-level |

Define ICP first. Persona lives inside the ICP.

---

## ICP statement structure

```
For [specific company type] that [are experiencing X situation/trigger],
[Product] is the [category] they use to [achieve outcome],
unlike [primary alternative] which [key shortcoming].
```

Example:
```
For B2B SaaS companies with 10–100 employees that are scaling their support team
past what spreadsheets can handle, [Product] is the customer data platform they
use to reduce ticket resolution time by 40%, unlike Zendesk which requires a
dedicated admin and 3-month onboarding.
```

## Attribute taxonomy

### B2B firmographic attributes

| Attribute | Why it matters |
|-----------|---------------|
| **Company size** (employees or ARR) | Determines budget, process maturity, and buy cycle length |
| **Industry / vertical** | Determines regulatory context, competitive set, and terminology |
| **Company stage** | Startup vs growth vs enterprise determines urgency and risk tolerance |
| **Tech stack** | Integration requirements; signals sophistication level |
| **Geography** | Data residency, compliance, support timezone, language |
| **Revenue model** | SaaS vs transactional vs services — affects their own customer relationships |

### B2B behavioral attributes

| Attribute | Signal source |
|-----------|-------------|
| **Current solution** | Are they using a competitor, a spreadsheet, or nothing? |
| **Team structure** | Do they have a dedicated person for this problem? |
| **Decision-making speed** | How long does their typical software buy take? |
| **Product adoption signals** | Do they use integrations? API? Power features? |

## Situational triggers

The trigger is the most important ICP attribute. It answers: **why do they buy now, not in 6 months?**

Common triggers by category:

| Trigger type | Example |
|-------------|---------|
| **Growth event** | Hired 5th salesperson; team doubled; passed $1M ARR |
| **Pain threshold** | Spreadsheet broke under load; existing tool too slow |
| **Compliance event** | SOC2 audit; GDPR requirement; board mandate |
| **Technology change** | Migrating off legacy system; existing vendor sunset |
| **Competitive pressure** | Competitor launched a feature they need to match |
| **New leader** | New VP of Sales/Marketing/Eng brings new tool preferences |
| **Failed DIY** | Built internal tool that is now a maintenance burden |

If you can name the trigger, you can find prospects experiencing it right now.

## Anti-ICP definition

Define the negative ICP explicitly to save sales and CS cycles:

| Anti-ICP attribute | Why they appear similar | Why they churn |
|-------------------|------------------------|---------------|
| Single-person companies | Look like SMB | No team to spread product; hit limits immediately |
| Highly regulated enterprise | High ACV looks attractive | Procurement takes 6+ months; requirements exceed product |
| Wrong industry vertical | Use same tech stack | Different terminology / workflow; heavy customization requests |
| DIY-first companies | High technical sophistication | Build instead of buy; low retention if product isn't 10× better |

## ICP scoring rubric

Use this to qualify or disqualify prospects quickly:

| Attribute | Score |
|-----------|-------|
| Company size in target range | +2 |
| Situational trigger present | +3 (highest weight) |
| Uses a tool you replace | +2 |
| Industry in target vertical | +1 |
| Decision-maker is the champion | +2 |
| Anti-ICP flag present | −4 |

**Score ≥ 7**: high priority
**Score 4–6**: qualify further
**Score < 4**: deprioritize

## Narrow-ICP-first strategy

Counter-intuitively, starting with a tighter ICP produces faster growth:

1. **Better product-market fit** — you can solve one segment's problem deeply instead of many segments' problems shallowly
2. **Cheaper CAC** — narrow targeting reduces wasted ad spend and sales cycles
3. **Higher NPS** — customers who are a perfect fit churn less and refer more
4. **Clearer positioning** — you can say something specific to a tight ICP; broad ICPs produce generic messaging

Expand the ICP only after achieving strong NRR (>110%) and NPS (>40) in the core segment.

## How ICP shapes everything else

| Function | ICP input |
|----------|-----------|
| **Pricing** | ICP company size and budget determines willingness to pay and tier structure |
| **Positioning** | ICP trigger and pain point determines what you say and where you say it |
| **Channel** | ICP watering holes (communities, events, publications) determine acquisition channel |
| **Onboarding** | ICP use case determines the activation milestone and email sequence |
| **Product roadmap** | ICP pain points determine which features to build next |
| **CS segmentation** | ICP ARR band determines high-touch vs low-touch CS model |
