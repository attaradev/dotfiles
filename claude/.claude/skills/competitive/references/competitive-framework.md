# Competitive Analysis Framework

## Player classification

Classify every player before analysing them:

| Type | Definition | Example |
|------|-----------|---------|
| Direct competitor | Same job, same target segment, similar approach | Another SaaS project management tool targeting SMB |
| Adjacent competitor | Same job, different segment or approach | Enterprise tool that could move down-market |
| Indirect substitute | Different approach to the same job | Email threads and spreadsheets instead of a dedicated tool |
| Non-consumption | The user does not solve the problem at all | Doing nothing; the status quo |

Non-consumption is always a competitor. Understanding why users choose not to solve the problem is often more revealing than analysing direct competitors.

## Analysis structure

### 1. Player map

List every relevant player with a one-sentence description of their positioning.

| Player | Type | Positioning | Primary segment | Business model |
|--------|------|------------|----------------|---------------|
| [Competitor A] | Direct | [Their core claim] | [Who they target] | SaaS / usage / freemium |

### 2. Positioning dimensions

Identify the two to four dimensions that matter most to users when choosing in this market. Common dimensions:

- Price / total cost of ownership
- Ease of use / time to value
- Depth of features
- Integration ecosystem
- Security and compliance
- Support and SLA
- Self-serve vs. sales-led
- Vertical specialisation

### 3. Feature matrix

Rate each player on the dimensions that matter. Use: ✓ (has it), ~ (partial), ✗ (missing), ? (unknown).

| Capability | Us | Competitor A | Competitor B | Competitor C |
|-----------|----|--------------|--------------| -------------|
| [Feature 1] | ✓ | ✓ | ~ | ✗ |
| [Feature 2] | ~ | ✓ | ✓ | ? |

Only include capabilities that users actually care about. A 40-row feature matrix where 35 rows are identical across all players obscures what matters.

### 4. Strengths and weaknesses

For each direct competitor, one paragraph each:

**Competitor A**
- Strongest at: [What they genuinely do better than anyone]
- Weakest at: [Where users consistently complain or switch away]
- Evidence: [Review quotes, job postings, public roadmap signals]

### 5. Gaps and opportunities

Identify where demand is underserved across the landscape — high importance to users, low satisfaction with current solutions.

| Gap | Who has it | Evidence | Opportunity size |
|-----|-----------|---------|-----------------|
| [Unmet need] | [Which players address it, how well] | [Review data, support tickets, user quotes] | H/M/L |

### 6. Differentiation angle

State the defensible position given this landscape:

**We win for [specific segment] on [specific dimension] because [specific reason].**

A good differentiation angle is:
- Specific enough to be falsifiable ("best for engineering teams of 10–50 who care about audit trails")
- Based on something real (a technical advantage, a distribution advantage, a data advantage, or deep segment expertise)
- Hard enough for competitors to replicate that it provides a durable advantage

## Research sources

Use these to gather evidence:

| Source | Best for |
|--------|---------|
| G2, Capterra, Trustpilot reviews | Real user pain points, switch reasons, comparison searches |
| App Store / Play Store reviews | Mobile UX, onboarding friction, feature requests |
| Competitor job postings | Investment areas, technical stack, go-to-market motion |
| Competitor engineering blogs | Technical architecture, scalability decisions |
| SEO keyword research | What problems users search for; what comparisons they make |
| Sales call notes | Objections, competitive displacement patterns |
| Churned customer interviews | Why they switched and to what |

## Positioning map template

Place each player on a 2×2 using the two most important dimensions for your market. This is a communication tool for alignment — choose dimensions that the audience understands and that reflect real trade-offs.

Example axes:
- X: Self-serve → Sales-led
- Y: Point solution → Platform

Label each quadrant and note which player owns each space. An empty or underserved quadrant may be an opportunity.
